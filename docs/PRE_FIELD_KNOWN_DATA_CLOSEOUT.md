# Pre-field known-data closeout

## Status

This repository has completed its **published/public known-data phase**. The
phase is frozen by `data/known_data_phase_lock.json` and can be rendered with:

```bash
python scripts/build_known_data_phase_closure.py \
  --repository-root . \
  --output artifacts/PRE_FIELD_KNOWN_DATA_CLOSURE.md
```

The lock preserves the retained source-row counts, six-candidate SMC snapshot,
sensitivity results, claim ledger, and field handoff. It is intentionally not
updated by adding another proxy model or by recoding a public-photo discovery
lead as a trait.

## What the known data support

1. The retained Inoue-series summaries contain coordinated island-associated
   differences in outcrossing, bagged capsule set, and common-garden flower
   length.
2. In the saved six-candidate compatibility screen, `isolation_order` has the
   highest mean compatibility. This identifies an unresolved island-associated
   gradient, not an isolation mechanism.
3. The declared `ardens_step_persistence` candidate is more compatible on
   average than the old continuous-current-pollinator `ardens_bridge_loss`
   candidate. Thus an initial Oshima-like size step remains a serious working
   explanation.
4. Flower length is the current principal discriminating channel: removing it
   changes the leading source-level candidate. This makes it valuable, but also
   means it cannot by itself settle history or mechanism.

## What the known data do not support

The phase lock explicitly blocks claims of:

- a proven causal role of *Bombus ardens* in floral-size evolution;
- a dated route of colonization, sequential speciation, or historical bee loss;
- small bees being irrelevant to later reproductive change;
- island-level nectar-guide loss, guide direction, or guide function;
- visitor absence, pollen deposition, pollinator effectiveness, or reproductive
  fitness from occurrence or visit descriptions alone.

The Hachijo inner-corolla text record is retained only as a discovery lead. It
is not an ordinal guide score, a prevalence estimate, an island contrast, or a
model constraint.

## Why this is a productive stopping point

The analyses did not merely fail to decide among stories. They identified why
the stories remain confounded:

- the ordinal island proxy can represent history, unmeasured environment,
  pollinator turnover, or any correlated gradient;
- the flower-length channel carries much of the current discrimination but has
  limited context structure;
- published visitor descriptions rarely measure organ contact, pollen transfer,
  or comparable observation effort;
- guide evidence has no reviewed island-resolved trait constraint.

Further variation of known-data proxy models would therefore risk repeating the
same non-identification. The next valid inference phase begins only when
first-party data add a genuinely new channel.

## Entry condition for the first-party phase

For a tagged plant, link the following before making an island comparison:

```text
field event -> island -> site -> plant -> flower
                     |          |        |
          inner-corolla image   geometry  observation effort + visit-bout handling
```

Use `docs/field_multichannel_coverage.md` to identify missing links. A visually
confirmed anther-plus-stigma contact is still only a handling proxy; pollen
deposition and reproductive outcome remain separate next-step channels.

## Transition rule

Any analysis after first-party data arrive must state whether it:

1. reproduces the frozen known-data result unchanged;
2. adds a predeclared new observation channel; or
3. revises the claim ledger with an explicit reason and separate output.

Do not silently replace the pre-field result register with a new field-informed
model result.
