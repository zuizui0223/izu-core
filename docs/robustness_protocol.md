# Pre-data robustness and operating-characteristic protocol

## Why this protocol exists

A deterministic reconstruction test is necessary but insufficient.  A scenario
can recover itself exactly when synthetic observations are generated by the
same equations and supplied with almost zero-width intervals, while still
failing in the field because of finite samples, multiple simultaneous
measurements, unrepresented compound mechanisms, or site-level confounding.

This protocol turns those failure modes into explicit pre-data tests.  It does
not validate a historical mechanism.  It asks whether a *declared field design*
would retain or distinguish a known virtual mechanism often enough to justify
collecting its measurements.

## Mandatory stress tests

### 1. Simultaneous-observation coverage

Do not treat several marginal 95% intervals as one joint 95% observation.  If
four observations are each checked separately, their joint coverage is usually
far below 95%.

Use `SimultaneousIntervalPlan` to declare a family-wise target and derive a
Bonferroni marginal confidence.  Then simulate the observation model and report
its realised joint coverage.  Bonferroni controls the union bound only when
individual intervals themselves have their intended coverage; an approximate
normal interval for sparse count data may not do so.  Replace the benchmark
normal interval with an assay-appropriate count or hierarchical observation
model before field-facing use.

```python
from channel_id.observation import SimultaneousIntervalPlan

plan = SimultaneousIntervalPlan(
    familywise_confidence=0.95,
    observation_count=4,
)
print(plan.describe())
# bonferroni family-wise 0.950; 4 observables; marginal confidence 0.987500
```

### 2. Compound-route recovery

A field mechanism need not be one named route.  In particular, guide-mediated
visitation and delayed selfing can operate together.  Test the candidate set
against at least the following maternal alternatives:

```text
null
visit attraction
handling
cost only
assurance
visit + assurance
handling + assurance
visit + cost
handling + cost
visit + handling
visit + handling + assurance
```

Use `GuideRoutes` for a particular declared combination and
`core_maternal_scenarios()` for this starting set.  Do not use the broad `mixed`
scenario as a shortcut for these combinations: `mixed` additionally activates
paternal and spatial pathways and should only be compared once their
intermediate observations exist.

### 3. Candidate-model misspecification

For every planned model family, simulate virtual truths that are deliberately
absent from a reduced candidate set.  Record whether recovery produces:

- a non-empty but misleading compatible set;
- an empty compatible set;
- a broad compatible set that honestly expresses ambiguity.

A high empty-set rate is a design warning, not evidence that the real system is
mechanistically exotic.  It can result from too-narrow intervals, an omitted
route, an inadequate observation model, or a data-quality problem.

### 4. Site and plant-condition confounding

A guide contrast that covaries with flower size, nectar, plant condition, site,
or observation time is not a causal guide exposure.  `ChannelDesign` now
reports a separate guide-causal status:

```text
association_only
conditional_within_site_contrast
ready_manipulated_contrast
```

For the first Izu field season, a natural contrast should be reported as
conditional unless it uses within-site or matched-population comparisons,
measures display/nectar/plant condition, and blocks time, flower age, and
weather.  A causal contrast additionally needs a guide manipulation with a
credible sham control.

### 5. Density-dependent recruitment

When seed supply may alter per-seed recruitment, compare a constant `E` with a
Beverton-Holt alternative:

```text
E(F) = E0 / (1 + F / H)
W = F E(F)
```

Use this only with seed-addition, seed-density, or cohort-follow-up data at a
common spatial and temporal census scale.  `BevertonHoltEstablishment` is a
minimal optional layer; it does not replace a spatial or seed-bank model where
those processes are measured.

## Finite-sample scenario benchmark

`channel_id.operating_characteristics` implements a lightweight starting
benchmark.  It samples individual-level Poisson counts around each selected
per-maternal expectation, turns them into simultaneous intervals, and recovers
all compatible scenarios.

```python
from channel_id.guide_scenarios import (
    GuideRoutes,
    GuideScenario,
    ScenarioMetric,
)
from channel_id.operating_characteristics import (
    FiniteSampleRecoveryDesign,
    benchmark_scenario_recovery,
)

truth = GuideRoutes("visit_assurance", visit_attraction=True, assurance=True)
candidates = (
    GuideScenario.NULL,
    GuideScenario.VISIT_ATTRACTION,
    GuideScenario.ASSURANCE,
    truth,
)
design = FiniteSampleRecoveryDesign(
    maternal_individuals=80,
    metrics=(
        ScenarioMetric.EXPECTED_VISITS,
        ScenarioMetric.OUTCROSS_VIABLE_SEEDS,
        ScenarioMetric.SELFED_VIABLE_SEEDS,
    ),
    familywise_confidence=0.95,
)
summary = benchmark_scenario_recovery(
    truth,
    candidates,
    settings,
    "typical",
    design,
    replicates=1_000,
    seed=20260629,
)
print(summary)
```

The summary reports:

- `truth_retained_rate`: how often the compatible set contains the virtual truth;
- `unique_truth_recovery_rate`: how often that truth is the only survivor;
- `empty_compatible_set_rate`: how often every candidate is rejected;
- `mean_compatible_scenarios`: residual ambiguity after measurement.

The included Poisson sampler is deliberately not a proposed likelihood for
field data.  Replace it with an observation model calibrated to the planned
assay: overdispersed visit counts, seed-set and paternity uncertainty, genotype
error, zero-inflated recruitment, site and year random effects, and incomplete
detection as appropriate.

## Acceptance standard before field-facing claims

For each intended Izu claim, predeclare:

1. virtual truths and plausible competing/compound routes;
2. the actual sample size, sites, maternal families, years, and observation
   windows planned;
3. the observation model and uncertainty intervals for each metric;
4. joint coverage under that model;
5. truth retention, unique recovery, and empty-set rates;
6. the sample-size or protocol change required if recovery is inadequate.

Only after this benchmark should a high-ranking measurement become a field
protocol.  A successful virtual recovery remains a design result, not evidence
that the same route caused the real island pattern.
